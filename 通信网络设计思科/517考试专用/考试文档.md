### 路由器R1上进行ACL包过滤配置

左侧PC0VLAN90、PC1VLAN190的PC只能访问总部服务器服务器0,PC2VLAN290的PC只能以WEB方式访问服务器1，不可访问10.91.1.0网段

#### 路由器R1配置

##### 定义扩展ACL(控制具体访问规则)

- ACL 100：允许 VLAN90/190 访问服务器 0，允许 VLAN290 以 WEB 访问服务器 1，不能访问10.91.1.0网段

```
R1>enable
R1#configure terminal

! 配置ACL 100（扩展ACL，编号100-199）配置顺序严格遵守
R1(config)#access-list 100 permit ospf any any       ! 允许OSPF协议（路由同步必备）
R1(config)#access-list 100 permit udp any any eq 67  ! 允许DHCP服务器端口（UDP 67）
R1(config)#access-list 100 permit udp any eq 67 any eq 68  ! 允许DHCP客户端响应

R1(config)#access-list 100 permit ip 10.90.1.0 0.0.0.255（PC0） host 10.91.2.1(服务器0)
#允许 ip网段+反子网掩码 访问 ip
R1(config)#access-list 100 permit ip 10.90.2.0 0.0.0.255（PC1） host 10.91.2.1(服务器0)
R1(config)#access-list 100 permit tcp 10.90.3.0 0.0.0.255（PC2） host 10.91.2.2 eq www (服务器1)


R1(config)#access-list 100 deny ip 10.90.1.0 0.0.0.255（PC0） host 10.91.2.2(服务器1)
R1(config)#access-list 100 deny ip 10.90.2.0 0.0.0.255（PC1） host 10.91.2.2(服务器1)
R1(config)#access-list 100 deny ip 10.90.3.0 0.0.0.255 （PC2）host 10.91.2.1(服务器0)
R1(config)#access-list 100 deny ip 10.90.3.0 0.0.0.255（PC2） host 10.91.2.2(服务器1)

#不允许网段+反子网掩码 访问 网段+反子网掩码
R1(config)#access-list 100 deny ip 10.90.1.0 0.0.0.255（PC0） 10.91.1.0 0.0.0.255 (PC4\PC5)
R1(config)#access-list 100 deny ip 10.90.2.0 0.0.0.255(PC1) 10.91.1.0 0.0.0.255  (PC4\PC5)
R1(config)#access-list 100 deny ip 10.90.3.0 0.0.0.255（PC2） 10.91.1.0 0.0.0.255  (PC4\PC5)

#其他无关流量可放行
R1(config)#access-list 100 permit ip any any


```

#### 路由器R3配置

##### 接口IP配置

```
R3>enable
R3#configure terminal
# 接口Serial0/0（公网侧，DTE端，不必配时钟频率）
R3(config)#interface Serial0/0
R3(config-if)#ip address 3.90.1.1 255.255.255.252
R3(config-if)#no keepalive
R3(config-if)#no shutdown
R3(config-if)#exit

#接口FastEthernet0/0（私网侧，连R1）
R3(config)#interface FastEthernet0/0
R3(config-if)#ip address 192.168.1.6 255.255.255.252
R3(config-if)#no shutdown
R3(config-if)#exit

#接口FastEthernet0/1（私网侧，连R2）
R3(config)#interface FastEthernet0/1
R3(config-if)#ip address 192.168.1.10 255.255.255.252
R3(config-if)#no shutdown
R3(config-if)#exit
```

##### 静态路由配置

```
#配置默认路由
R3(config)#ip route 0.0.0.0 0.0.0.0 3.90.1.2#0.0.0.0 0.0.0.0 +下一跳
R3(config)#exit
```

##### R3下发默认路由到私网路由器

```
R3(config)#
#启用OSPF（进程号1，与私网设备一致）
R3(config)#router ospf 1
#宣告私网互联网段（连R1的192.168.1.4/30）
R3(config-router)#network 192.168.1.4 0.0.0.3 area 0
#宣告私网互联网段（连R2的192.168.1.8/30）
R3(config-router)#network 192.168.1.8 0.0.0.3 area 0
#核心：向私网发布默认路由（仅私网路由器可见）
R3(config-router)#default-information originate
# 拒绝将公网静态路由注入OSPF（避免扩散到私网）
R3(config-router)#no redistribute static subnets
R3(config-router)#exit
```

### 在私网与公网的边界路由器R3上进行PAT配置

![image-20251123002821205](C:\Users\86185\AppData\Roaming\Typora\typora-user-images\image-20251123002821205.png)

使得PC4、PC5可以访问公网，左侧都只有vlan90的PC可以访问公网1

#### 路由器R3配置

```
R3(config)#access-list 101 permit ospf any any
R3(config)#access-list 101 permit ip 10.90.1.0 0.0.0.255(PC0) 11.90.1.0 0.0.0.255（PC9）
R3(config)#access-list 101 permit ip 10.91.1.0 0.0.0.255（PC4\PC5） 11.90.1.0 0.0.0.255 (PC9)
R3(config)#access-list 101 deny ip 10.91.2.0 0.0.0.255（服务器0、1） 11.90.1.0 0.0.0.255（PC9）
R3(config)#access-list 101 deny ip 10.90.2.0 0.0.0.255（PC1） 11.90.1.0 0.0.0.255 (PC9)
R3(config)#access-list 101 deny ip 10.90.3.0 0.0.0.255(pC2) 11.90.1.0 0.0.0.255 (PC9)
R3(config)#access-list 101 permit ip any any

# 3. 配置PAT（Easy IP模式，绑定公网接口IP）
R3(config)#ip nat inside source list 101 interface Serial0/0 overload
# 注：overload实现多私网IP共享一个公网IP，即PAT核心功能

! 4. 标记NAT内外网接口（关键：区分流量方向）
R3(config)#interface FastEthernet0/0
R3(config-if)#ip nat inside
R3(config-if)#exit
! 私网接口Fa0/1（左侧）标记为inside
R3(config)#interface FastEthernet0/1
R3(config-if)#ip nat inside
R3(config-if)#exit
! 公网接口Serial0/0（转换侧）标记为outside
R3(config)#interface Serial0/0
R3(config-if)#ip nat outside
R3(config-if)#ip access-group 101 out
R3(config-if)#exit
```

### 仅允许10.91.1.0能够telnet到三层交换机S3

#### 交换机S3配置

##### 配置telnet登录

```
S3(config)# line vty 0 4  // 进入VTY线路（0-4共5个会话）
S3(config-line)# login local  // 启用本地用户认证
S3(config-line)# transport input telnet  // 允许Telnet协议（默认可能已开启）
S3(config-line)# exit
```

##### 创建本地用户

```
S3(config)# username admin password 123456  // 创建用户admin，密码123456
```

##### 配置ACL允许10.91.1.0访问

```
S3(config)# access-list 10 permit 10.91.1.0 0.0.0.255(pc 4\pc5)  // 创建ACL 10，允许10.91.1.0/24网段
S3(config)# line vty 0 4
S3(config-line)# access-class 10 in  // 将ACL 10应用到VTY线路的入方向（仅允许ACL匹配的地址访问）
S3(config-line)# exit
```
